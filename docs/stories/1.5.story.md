# Story 1.5: Classificador de Intencao (Haiku)

## Objetivo

Implementar o classificador de intencao que usa Claude Haiku 4.5 para identificar o que o aluno quer em cada mensagem. O classificador distingue entre 14 intencoes diferentes (saudacao, continuar aula, enviar codigo, duvida, etc.) e e essencial para que o Koda responda de forma contextual. Sem ele, o Koda nao sabe se o Pedro quer continuar a aula ou tirar uma duvida.

## Acceptance Criteria

- [x] AC1: `src/core/classifier.ts` exporta funcao `classifyIntent(message: string, currentState: string): Promise<ClassificationResult>`
- [x] AC2: Interface `ClassificationResult` definida com: `intent: IntentType`, `confidence: number` (0.0 a 1.0)
- [x] AC3: Tipo `IntentType` definido como union de 14 strings: `greeting`, `onboarding_response`, `lesson_continue`, `lesson_explain_again`, `code_submission`, `exercise_answer`, `gate_response`, `doubt`, `progress_check`, `mood_check`, `quiz_answer`, `off_topic`, `audio`, `image`
- [x] AC4: Prompt de classificacao criado em `prompts/classifier.md` com instrucoes claras, lista de intencoes com exemplos, e formato de saida JSON `{ "intent": "...", "confidence": 0.0-1.0 }`
- [x] AC5: Classificador usa Claude Haiku (`claude-haiku-4-5-20250514`) via `src/services/claude.ts`
- [x] AC6: O prompt inclui o `currentState` da conversa para contextualizacao (ex: se esta em GATE_1, uma explicacao provavelmente e `gate_response`, nao `lesson_continue`)
- [x] AC7: Se a confianca for < 0.5, retorna intent `unknown` em vez do intent classificado
- [x] AC8: Se a chamada ao Claude falhar (timeout, erro de API), retorna fallback `{ intent: "unknown", confidence: 0 }` sem quebrar o pipeline
- [x] AC9: Latencia da classificacao logada: `Intent classified as {intent} (confidence: {X}) in {Y}ms`
- [x] AC10: Teste unitario com mock do Claude para pelo menos 5 cenarios: saudacao ("oi"), continuacao ("vamos la"), duvida ("o que e variavel?"), codigo ("function soma(){}"), humor ("cansei")

## Technical Notes

- Usar modelo `claude-haiku-4-5-20250514` — rapido (~200-500ms) e barato (~$0.001/classificacao)
- O prompt deve ser conciso para minimizar tokens e custo
- O `currentState` e crucial: "sim" pode ser `onboarding_response` (em ONBOARDING), `lesson_continue` (em HUB), ou `gate_response` (em GATE_1)
- O classificador e chamado na Etapa 4 do pipeline (Story 1.4)
- Referencia de intencoes: `docs/prd.md` secao FR2 e `docs/architecture/fullstack-architecture.md` secao 5
- Nao usar tools/function calling — apenas prompt com instrucao de retornar JSON
