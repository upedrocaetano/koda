# Story 2.3 — Autenticação (Supabase Auth)

## Objetivo

Implementar autenticação completa no Web App usando Supabase Auth. Cadastro/login com email+senha ou Google OAuth. Rotas protegidas redirecionam para login. Hook `useAuth()` para estado de autenticação.

## Acceptance Criteria

- [x] AC1: Supabase client utilities criadas (server + browser) usando `@supabase/ssr`
- [x] AC2: Página `/login` com formulário Matrix-themed: email, senha, botão "Entrar", botão "Google", link "Criar conta"
- [x] AC3: Página `/signup` com formulário: nome, email, senha, confirmação. Botão "Criar conta", link "Já tenho conta"
- [x] AC4: Google OAuth: botão inicia fluxo OAuth, callback em `/auth/callback` processa token e redireciona para `/dashboard`
- [x] AC5: Middleware protege rotas `/dashboard/*`, `/chat/*`, `/profile/*`, `/lessons/*`. Redireciona para `/login` se não autenticado
- [x] AC6: Hook `useAuth()` retorna: `user`, `profile`, `isLoading`, `isAuthenticated`, `signOut()`
- [x] AC7: Após login, redireciona para `/dashboard`. Após logout, para `/login`
- [x] AC8: Sessão persiste entre refreshes via cookies

## Technical Notes

- `@supabase/ssr` para Next.js App Router (não `@supabase/auth-helpers`)
- Google OAuth requer configuração no dashboard Supabase
- Middleware roda no edge — usar `createServerClient` do `@supabase/ssr`
- RLS policies do Epic 1 já existem
