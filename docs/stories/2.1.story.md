# Story 2.1 — Setup Monorepo & Scaffold Next.js

## Objetivo

Reestruturar o projeto como monorepo com Turborepo, extraindo módulos reutilizáveis do Epic 1 para `packages/` e criando o scaffold da aplicação Next.js em `apps/web`.

## Acceptance Criteria

- [x] AC1: `turbo.json` configurado na raiz com tasks: `build`, `dev`, `lint`, `typecheck`, `test`
- [x] AC2: Workspace structure: `apps/whatsapp` (backend Hono movido), `apps/web` (Next.js), `packages/shared`, `packages/db`, `packages/gamification`
- [x] AC3: `packages/shared` contém logger, tipos compartilhados, constantes
- [x] AC4: `packages/db` contém schema.ts, queries/, cliente Supabase configurável
- [x] AC5: `packages/gamification` contém xp-calculator.ts e streak-tracker.ts (funções puras)
- [x] AC6: `apps/whatsapp` funciona identicamente ao backend atual — todos os testes passam
- [x] AC7: `apps/web` criado com Next.js (App Router), TypeScript strict, Tailwind CSS
- [x] AC8: `apps/web` importa tipos e funções de `@koda/shared`, `@koda/gamification`, `@koda/db`

## Technical Notes

- npm workspaces (não pnpm — manter consistência com Epic 1)
- Cada package tem `package.json` com `name: "@koda/..."`
- Funções puras vão para packages, funções com side-effects ficam em apps ou packages/db
- Next.js usa App Router por default
