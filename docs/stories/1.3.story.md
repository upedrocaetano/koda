# Story 1.3: Integracao Webhook Evolution API

## Objetivo

Criar a rota que recebe mensagens do WhatsApp via Evolution API e envia respostas de volta. Este e o ponto de entrada de TODA comunicacao do Koda — sem ele, Pedro nao consegue conversar com o bot. A integracao deve ser robusta: lidar com mensagens duplicadas (idempotencia), diferentes tipos de midia e validacao de seguranca.

## Acceptance Criteria

- [x] AC1: Rota `POST /webhook/evolution` criada em `src/routes/webhook.ts` e registrada no Hono app
- [x] AC2: Rota filtra apenas eventos `messages.upsert` — outros eventos retornam `{ ok: true }` sem processamento
- [x] AC3: Rota ignora mensagens enviadas pelo proprio bot (campo `key.fromMe === true`)
- [x] AC4: Parsing correto do payload: extrai `phone` (remove `@s.whatsapp.net`), `text` (de `conversation` ou `extendedTextMessage.text`), `messageId` (de `key.id`)
- [x] AC5: Deteccao de tipo de mensagem: `text`, `audio` (audioMessage), `image` (imageMessage) — armazena tipo em variavel `messageType`
- [x] AC6: Idempotencia implementada: verifica se `messageId` ja foi processado (cache in-memory com TTL de 5 minutos). Se duplicado, retorna `{ ok: true, duplicate: true }` sem reprocessar
- [x] AC7: `src/services/evolution.ts` exporta funcao `sendMessage(phone: string, text: string): Promise<void>` que envia mensagem via Evolution API (`POST /message/sendText/{instance}`)
- [x] AC8: `src/services/evolution.ts` exporta funcao `sendTypingIndicator(phone: string): Promise<void>` que envia indicador "digitando..." via Evolution API
- [x] AC9: Validacao de API key: webhook valida header `apikey` contra `EVOLUTION_WEBHOOK_SECRET` env var. Se invalido, retorna 401
- [x] AC10: Rota retorna 200 imediatamente e processa mensagem de forma assincrona (nao bloqueia o webhook)
- [x] AC11: Tratamento de erro: se processamento falha, loga o erro mas NAO retorna erro para o Evolution (evita retry infinito)
- [x] AC12: Teste unitario para parsing de payload com pelo menos 3 cenarios: mensagem de texto simples, mensagem com formatacao (extendedTextMessage), mensagem de audio

## Technical Notes

- Evolution API ja esta rodando na VPS do Pedro — URL e porta configuradas via `EVOLUTION_API_URL`
- Instancia configurada via `EVOLUTION_INSTANCE` (provavelmente "koda")
- API key da Evolution via `EVOLUTION_API_KEY` (para enviar mensagens)
- Webhook secret via `EVOLUTION_WEBHOOK_SECRET` (para validar webhooks recebidos)
- Referencia de payload: `docs/architecture/fullstack-architecture.md` secao 9
- O `sendMessage` deve quebrar textos > 2000 caracteres em multiplas mensagens (limite pratico do WhatsApp)
- O cache de idempotencia pode ser um `Map<string, number>` com limpeza periodica — suficiente para MVP
