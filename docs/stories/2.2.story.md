# Story 2.2 — Matrix Design System

## Objetivo

Implementar o Design System Matrix como base visual do Web App. Configurar fontes adicionais (Inter, Fira Code), componente de Code Rain canvas, e componentes base reutilizáveis (Button, Card, Input, Badge, ProgressBar). Todo componente segue o padrão Matrix: fundo escuro, texto verde fosforescente, efeitos de glow.

## Acceptance Criteria

- [x] AC1: `globals.css` atualizado com `@font-face` / `next/font` para Inter e Fira Code. Tokens CSS já configurados na Story 2.1
- [x] AC2: Componente `CodeRainCanvas` implementado como canvas animado de caracteres verdes caindo. Props: `opacity`, `speed`, `density`. Respeita `prefers-reduced-motion`
- [x] AC3: Componente `MatrixButton` com variantes: `primary`, `secondary`, `danger`, `ghost`. Efeito glow no hover. Estado `loading` com spinner
- [x] AC4: Componente `MatrixCard` com fundo `matrix-bg-card`, borda sutil, cantos arredondados, efeito scanline opcional. Props: `header`, `children`, `footer`
- [x] AC5: Componente `MatrixInput` com fundo `matrix-bg-input`, borda verde, texto verde. Suporta tipos text/email/password/number. Estado de `error`
- [x] AC6: Componente `MatrixBadge` para badges/tags. Variantes: `xp` (verde), `level` (dourado), `streak` (ciano), `error` (vermelho). Props: `icon`, `label`
- [x] AC7: Componente `MatrixProgressBar` com barra verde animada. Props: `value` (0-100), `label`, `showPercentage`. Glow no topo da barra
- [x] AC8: Todos os componentes exportados de `apps/web/components/ui/` com barrel export

## Technical Notes

- Usar `next/font` para Inter e Fira Code com `display: swap`
- Code Rain usa canvas 2D — não WebGL
- Usar `cva` (class-variance-authority) para variantes de componentes
- Componentes são Server Components por default, Client Components apenas quando necessário
- `prefers-reduced-motion` desativa Code Rain e animações de glow
