# Story 1.7: Fluxo de Onboarding

## Objetivo

Implementar o fluxo de onboarding de 4 perguntas que coleta informacoes do aluno na primeira interacao. Pedro sera o primeiro a passar por esse fluxo — o Koda vai se apresentar, perguntar nome, objetivo, nivel e disponibilidade, e criar o perfil completo no Supabase. E a primeira impressao do Pedro com o Koda, entao precisa ser acolhedora e TDAH-friendly (rapida, sem burocracia).

## Acceptance Criteria

- [x] AC1: `src/modules/onboarding/handler.ts` exporta funcao `handleOnboarding(user: User, message: string, step: number): Promise<OnboardingResult>`
- [x] AC2: Interface `OnboardingResult` definida com: `responseText: string`, `nextStep: number | null` (null = onboarding completo), `userData: Partial<User>` (dados coletados nesse step)
- [x] AC3: **Step 0 (apresentacao)**: quando usuario novo envia primeira mensagem, Koda se apresenta com mensagem acolhedora em portugues e pergunta o nome. Mensagem inclui tom amigavel, emojis, e mencao ao XP ("ate dizer cansei da XP")
- [x] AC4: **Step 1 (nome)**: Koda salva o nome do usuario e pergunta o objetivo. Opcoes apresentadas: "Aprender do zero", "Mudar de carreira", "Criar meu proprio app/SaaS"
- [x] AC5: **Step 2 (objetivo)**: Koda salva objetivo (mapeado para `zero_to_dev` | `career_change` | `create_saas`) e pergunta nivel. Opcoes: "Nunca programei", "Sei um pouco de HTML", "Ja sei JavaScript"
- [x] AC6: **Step 3 (nivel)**: Koda salva nivel (mapeado para `beginner` | `basic_html` | `knows_js`) e pergunta disponibilidade. Opcoes: "5-10 min/dia", "15-30 min/dia", "30+ min/dia"
- [x] AC7: **Step 4 (disponibilidade)**: Koda salva disponibilidade (mapeado para minutos: 10, 20, 45) e envia mensagem de conclusao com resumo do perfil e convite para primeira aula
- [x] AC8: Apos step 4, campo `onboarding_completed` do usuario e atualizado para `true` no Supabase
- [x] AC9: Apos step 4, estado da FSM transiciona de `ONBOARDING` para `HUB`
- [x] AC10: Prompt de onboarding criado em `prompts/onboarding.md` com personalidade do Koda, tom em portugues brasileiro, emojis, e instrucoes para cada step
- [x] AC11: `src/db/queries/users.ts` atualizado com funcao `updateUserProfile(userId: string, data: Partial<User>): Promise<void>` para salvar dados coletados a cada step
- [x] AC12: Se usuario ja completou onboarding e manda "oi", NAO repete onboarding — vai direto para HUB
- [x] AC13: Se usuario esta no meio do onboarding e manda mensagem inesperada (ex: "o que e programacao?"), Koda responde brevemente e redireciona para a pergunta atual
- [x] AC14: Registro de `conversation_state.context` atualizado a cada step com `{ onboarding_step: N }` para persistir progresso do onboarding entre mensagens

## Technical Notes

- O onboarding usa o AI Engine (Claude Sonnet) para gerar respostas naturais — nao e texto hardcoded
- O prompt em `prompts/onboarding.md` deve incluir as perguntas, opcoes esperadas e tom do Koda
- O handler e chamado pelo pipeline quando o estado e `ONBOARDING`
- Cada step atualiza o `context` da conversation_state para rastrear em qual pergunta esta
- A mensagem de apresentacao (step 0) deve seguir o tom definido em `docs/architecture/founder-first-strategy.md` secao "Mensagem do Koda para Pedro"
- Referencia do fluxo: `docs/prd.md` secao FR4 e User Flow 1
