# Story 1.1: Setup do Projeto & Infraestrutura

## Objetivo

Criar a base do projeto Koda — estrutura de pastas, configuracoes TypeScript, servidor Hono e cliente Supabase. Sem isso, nenhuma outra story pode comecar. Pedro precisa de um backend funcional rodando na VPS para que o Koda exista.

## Acceptance Criteria

- [x] AC1: `package.json` criado com dependencias: `hono`, `@hono/node-server`, `@supabase/supabase-js`, `@anthropic-ai/sdk`, `dotenv`, `tsx` (dev), `typescript` (dev)
- [x] AC2: `tsconfig.json` configurado com `strict: true`, `target: ES2022`, `module: NodeNext`, `moduleResolution: NodeNext`, paths com absolute imports
- [x] AC3: Estrutura de pastas criada conforme arquitetura: `src/routes/`, `src/core/`, `src/modules/`, `src/services/`, `src/db/`, `src/utils/`, `prompts/`, `curriculum/`
- [x] AC4: `src/index.ts` inicia servidor Hono na porta 3333 (configuravel via `PORT` env var)
- [x] AC5: Rota `GET /health` retorna `{ status: "ok", timestamp: "..." }` com status 200
- [x] AC6: `src/services/supabase.ts` exporta cliente Supabase configurado com `SUPABASE_URL` e `SUPABASE_SERVICE_ROLE_KEY` de env vars
- [x] AC7: `src/services/claude.ts` exporta cliente Anthropic configurado com `ANTHROPIC_API_KEY` de env var
- [x] AC8: `.env.example` criado com todas as variaveis necessarias documentadas (sem valores reais)
- [x] AC9: `.gitignore` inclui `node_modules/`, `.env`, `dist/`
- [x] AC10: `npm run dev` inicia o servidor com hot-reload (via `tsx watch`)
- [x] AC11: `npm run build` compila TypeScript sem erros
- [x] AC12: `src/utils/logger.ts` implementa logging estruturado com nivel (info, warn, error) e timestamp

## Technical Notes

- Entry point: `src/index.ts` — servidor Hono com `@hono/node-server`
- Usar `serve()` do `@hono/node-server` para rodar em Node.js (nao edge)
- Scripts do package.json: `dev` (tsx watch), `build` (tsc), `start` (node dist/index.js)
- O Supabase client usa `service_role_key` no backend (bypassa RLS) — nunca expor essa chave
- O cliente Claude sera usado tanto para Sonnet (ensino) quanto Haiku (classificador)
- Estrutura segue exatamente o diagrama em `docs/architecture/fullstack-architecture.md` secao 3
