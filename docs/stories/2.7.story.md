# Story 2.7 — Onboarding Web

## Objetivo

Criar o fluxo de onboarding de 4 etapas para novos usuarios do Web App. Telas visuais com animacoes Matrix, opcoes clicaveis, barra de progresso, e transicoes suaves. Coleta nome, objetivo, nivel e disponibilidade com UX rica.

## Acceptance Criteria

- [x] AC1: Pagina `/onboarding` renderizada apos primeiro login se `onboarding_completed === false`. Middleware redireciona `/dashboard` para `/onboarding`
- [x] AC2: Layout de onboarding: fundo escuro com Code Rain (opacidade 0.1), card central (max-width 480px), barra de progresso (4 etapas), botoes Voltar/Proximo
- [x] AC3: Etapa 1 — Boas-vindas: animacao typing "Bem-vindo ao Koda", campo de nome, validacao min 2 chars
- [x] AC4: Etapa 2 — Objetivo: 3 cards clicaveis (zero_to_dev, career_change, create_saas) com borda verde glow
- [x] AC5: Etapa 3 — Nivel: 3 cards clicaveis (beginner, basic_html, knows_js)
- [x] AC6: Etapa 4 — Disponibilidade: 3 cards clicaveis (10, 20, 45 minutos)
- [x] AC7: Ao concluir, salvar no Supabase, marcar `onboarding_completed = true`, animacao de transicao, redirect para `/dashboard`
- [x] AC8: Dados parciais persistidos; se usuario sair no meio, continua da etapa onde parou

## Dependencies

- Story 2.2 (Matrix Design System) ✅
- Story 2.3 (Supabase Auth) ✅
