# Story 2.6 — Fluxo de Aula (Lesson Flow)

## Objetivo

Implementar a experiencia de aula no Web App dentro da interface de chat. Quando Pedro inicia uma aula, o Koda ensina via chat com UI enriquecida: explicacoes com markdown formatado, exercicios com campos de codigo, Portao 1 (compreensao) com area de texto, Portao 2 (pratica) com editor inline, feedback visual de aprovacao/reprovacao, e XP ganho animado.

## Acceptance Criteria

- [x] AC1: Quando o estado da FSM e `LESSON`, o chat exibe um header contextual: nome do modulo, conceito atual, icone de progresso
- [x] AC2: Componente `GateCard` para Portao 1 (compreensao): card especial com titulo, descricao, textarea, botao "Enviar Resposta". Borda dourada
- [x] AC3: Componente `GateCard` para Portao 2 (pratica): card com titulo, descricao, editor de codigo inline, botao "Executar e Enviar". Borda dourada
- [x] AC4: Feedback de aprovacao: animacao de "PORTAO APROVADO" com glow verde, XP ganho animado
- [x] AC5: Feedback de reprovacao: mensagem encorajadora, borda vermelha sutil, contador de tentativas
- [x] AC6: Progresso da aula atualizado em tempo real no Supabase
- [x] AC7: Transicao entre estados (LESSON -> GATE_1 -> GATE_2 -> HUB) gerenciada pela FSM
- [x] AC8: Card de "Conceito Concluido" com resumo, XP total, proximo conceito sugerido

## Dependencies

- Story 2.5 (Chat Interface) ✅
